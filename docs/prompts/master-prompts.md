# Готовые промты для мастер-агента

Этот файл — библиотека готовых текстов, которые можно отправить мастеру через Telegram (`@master ...`) или вставить прямо в его tmux-окно. Мастер уже знает про AG-OS CLI и vault через скиллы в `.claude/skills/agos-*`, но конкретные формулировки часто удобнее держать под рукой.

## Управление агентами

### Создать нового агента для задачи X

```
@master создай sub-агента для <описание задачи>. Выбери подходящий runtime (docker если нужна изоляция) и провайдер из config.model_providers. Запиши его роль в private memory. Покажи финальный статус.
```

### Показать сводку по всем агентам

```
@master сделай человекочитаемую сводку всех активных агентов: имя, runtime, статус, провайдер, чем занимается (из current_task и memory). Выведи в виде таблицы.
```

### Пересоздать агента с новой моделью

```
@master пересоздай агента <name>, сменив провайдера на <provider>. Сначала проверь не осталось ли у него важного состояния в workspace или raw/<name>/ — если да, предупреди меня до destroy.
```

### Удалить всех dynamic агентов

```
@master найди всех агентов с type=dynamic и statusом kромe working — удали их. Оставь только permanent. Перед удалением покажи список кого собираешься убить.
```

## Планировщик и cron

### Поставить агента на cron

```
@master поставь агента <name> на расписание <cron|человеческое описание>. Промт должен быть таким: "<текст>". Сначала убедись что агент существует и у него есть нужные инструменты. Покажи созданную задачу через schedule list.
```

### Показать что в расписании

```
@master покажи расписание в читаемом виде: кто, когда, что делает, когда был последний запуск и с каким результатом. Если где-то есть повторяющиеся ошибки — подсветь.
```

### Остановить все cron-задачи агента

```
@master найди в расписании все задачи с agent=<name> и удали их. Покажи список перед удалением.
```

### Ручной запуск задачи

```
@master запусти задачу #<id> сейчас вручную, не дожидаясь cron.
```

## Obsidian vault

### Быстро записать идею

```
@master запиши в vault идею: <текст идеи>. Положи в raw/master/ с читаемым именем файла и коротким описанием. В конце напомни, что она попадёт в wiki после следующего processing-тика (или могу сейчас обработать, если скажешь).
```

### Обработать raw прямо сейчас

```
@master обработай свежие файлы во всех raw/*/ (кроме archive/). Для каждого реши: курировать в wiki или выбросить в архив. Новые wiki-страницы делай с полным фронтматтером и хотя бы одним [[link]] на существующую страницу. В конце сделай git commit в vault.
```

### Сводка по raw-блокнотам за сегодня

```
@master пройди по raw/*/ и покажи что агенты написали за последние 24 часа. Сгруппируй по агенту, кратко опиши каждую запись (тема + 1-2 предложения). Не обрабатывай — только обзор.
```

### Создать wiki-страницу про проект

```
@master создай или обнови wiki-страницу про проект <name>. Собери контекст: из моих сообщений, из raw/master/, из существующих wiki-заметок (поищи связанные). Структура: обзор, ключевые компоненты, связанные заметки, текущий статус. Фронтматтер полный.
```

### Построить backlinks

```
@master пройди по всем wiki-страницам, найди упоминания <тема> которые ещё не оформлены как [[link]], и добавь ссылки. Показывай diff перед каждым изменением.
```

### Найти что-то в vault

```
@master поищи в vault всё про <тема>. Посмотри и в raw (свежие мысли) и в wiki (уже курированные). Верни список файлов с кратким описанием релевантности каждого.
```

## Комбинированные сценарии

### Новый агент + cron + роль в памяти + заметка в vault

```
@master создай агента <name> на <provider> для задачи <описание>. Шаги:
1. Создай агента (docker-runtime)
2. Запиши в его private memory его роль и цели
3. Поставь на cron <расписание> с промтом "<текст>"
4. Создай в wiki/agents/<name>.md описание: зачем нужен, как настроен, что делает
Каждый шаг — короткое подтверждение, финально — сводка.
```

### Ежедневный дайджест

```
@master сделай дайджест за сегодня:
1. Что делали агенты (schedule list — какие задачи исполнились, как, результаты)
2. Что новое в raw/*/ (свежие заметки агентов)
3. Что нового в wiki/ (что ты обновил за сутки)
4. Есть ли ошибки или подозрительное
Дайджест положи в journal/YYYY-MM-DD.md в vault и сообщи мне короткую версию.
```

### Ревизия vault

```
@master сделай аудит vault:
1. Сколько файлов в raw/ (по агентам)
2. Сколько в wiki/ (по подпапкам)
3. Какие wiki-страницы не обновлялись больше месяца
4. Где в wiki нет backlinks (сиротские страницы)
5. Размер архива и сколько дней ретенции осталось
Коротко — что требует внимания.
```

### Очистка системы

```
@master наведи порядок:
1. Удали всех dynamic-агентов кроме тех, где statusом working
2. Удали из расписания задачи с last_result=error повторяющиеся 3+ раза подряд (подозрение на мёртвое)
3. Почисти raw archive старше raw_retention_days
4. Сделай git commit в vault
Перед каждым destructive-шагом — показывай что собираешься удалить.
```

## Правила использования

- **Всегда уточняй** у мастера, если задача размыта. Промт «создай агента» без контекста — плохой промт. Лучше: «создай sub-агента для чтения почты с mcp-gmail, на zai-glm, с cron каждый час».
- **Destructive действия** (`destroy`, `rm`, git операции) — проси мастера показать что он собирается сделать ДО того как он это сделает.
- **После сложных операций** — проси показать результат через `list`-команды.
- **Для регулярных задач** — вместо повторения одного и того же промта, лучше попросить мастера положить их в cron (см. раздел «Планировщик»).

## Где лежат скиллы

Мастер подтягивает инструкции автоматически из `.claude/skills/agos-*`. Если хочешь отредактировать поведение мастера — правь эти файлы, не промты выше. Файлы промтов — это шаблоны для человека, скиллы — для модели.
