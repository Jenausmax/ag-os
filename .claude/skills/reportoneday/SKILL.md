# Отчёт работ Green Team за день

Этот скилл собирает из Jira все записи рабочего времени (worklog) участников команды Green Team за указанную дату и формирует структурированный markdown-отчёт.

## Источник данных

Используй MCP-сервер `mcp-atlassian` для получения данных из Jira. Все запросы worklog идут через него.

## Алгоритм работы

### 1. Уточни дату

Спроси у пользователя, на какое число нужен отчёт, если он не указал дату явно. Если пользователь говорит "за сегодня" — используй текущую дату.

### 2. Собери записи времени

Для каждого участника команды найди все worklogs за указанную дату. Участники могут записывать время не только в задачи своей команды — ищи по автору worklog, а не по принадлежности задачи.

### 3. Получи информацию о задачах

Для каждой задачи, в которую записано время, получи:
- Ключ задачи (например CON-11709)
- Название задачи
- Текущий статус
- Assignee

### 4. Сформируй отчёт

Сохрани отчёт в файл `doc/reports/YYYY-MM-DD-green-team-отчет-работ.md` (дата в формате ISO).

Отчёт должен точно следовать структуре и формату из файла `references/example-report.md`. Прочитай его перед формированием отчёта, чтобы воспроизвести формат один в один.

## Структура отчёта (краткое описание секций)

1. **Заголовок** — `# Green Team — Отчёт работ за DD.MM.YYYY`
2. **Все записи времени** — две таблицы (Разработка и QA/Тестирование) с колонками: Участник, Задача, Время, Комментарий
3. **Итого** — сводная таблица по направлениям (Backend, Frontend, QA, Всего)
4. **По участникам** — таблица с суммарным временем каждого участника
5. **Кто НЕ записал время** — список участников без worklog за этот день
6. **Задачи дня — детали** — подробная информация по каждой задаче, разделённая на Разработка и QA

## Важные правила

- Давидовский Никита — бекендер, но если задача имеет префикс "Front:" в названии, всё равно относи его worklog к Backend в итогах (считаем по роли человека, а не по типу задачи)
- Время отображай в формате "Xч", "Xч Yм", "Yм" — без десятичных дробей
- В секции "Задачи дня — детали" для задач разработки добавляй префикс Back:/Front: к заголовку в зависимости от типа задачи. Для QA-задач префикс не нужен
- Если несколько человек записали время в одну задачу — в деталях указывай всех исполнителей через запятую
- Комментарии из worklog включай только если они не пустые
- Тимлид (Минаев Максим) обычно не записывает время — это нормально, просто отмечай его в секции "Кто НЕ записал время"
- В таблице "По участникам" сортируй по ролям: сначала Backend, потом Frontend, потом QA
