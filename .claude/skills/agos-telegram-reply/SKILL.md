# AG-OS: отвечать пользователю через Telegram

Этот скилл объясняет как мастер-агент возвращает ответы обратно в Telegram-чат пользователя. Без него бот работает односторонне — пользователь видит «Промт отправлен», но никаких ответов.

## Ключевой инвариант

**Каждое сообщение, пришедшее из Telegram, начинается с префикса:**

```
[ag-os chat=249402107 user=Max] текст от пользователя
```

- `chat=249402107` — это `chat_id`, который нужно передать в tool.
- `user=Max` — для твоей ориентации, в tool не нужен.
- Всё что после закрывающей `]` — реальный промт пользователя.

**Правило без исключений:** если видишь такой префикс — НЕ печатай ответ как обычный текст. Вызови tool `telegram_reply(chat_id=<тот самый id>, text="<твой ответ>")`. Обычный текстовый output из pane в Telegram **не попадает** — только явный вызов tool.

## Tool `telegram_reply`

Предоставляется MCP-сервером `ag-os-telegram`, который запускается автоматически вместе с тобой. Сигнатура:

```python
telegram_reply(chat_id: int, text: str) -> {ok: bool, message_id: int}
```

- `chat_id` — извлечённый из префикса.
- `text` — твой ответ. Plain text, Markdown не поддерживается в этой версии. Многострочный — можно.
- Возврат — факт доставки. Если `ok: False` или exception — Telegram API отклонил сообщение; залогируй причину и попробуй укоротить/упростить text.

## Типичные сценарии

### Сценарий 1: короткий ответ на простой вопрос

Входящий: `[ag-os chat=249402107 user=Max] привет, ты живой?`

```
telegram_reply(chat_id=249402107, text="Да, живой. Чем могу помочь?")
```

### Сценарий 2: выполнить команду и отчитаться

Входящий: `[ag-os chat=249402107 user=Max] покажи какие у нас агенты`

1. Выполни через Bash: `python main.py agent list`
2. Распарси вывод, составь человекочитаемую сводку.
3. Отправь результат:

```
telegram_reply(chat_id=249402107, text="Активных агентов: 2\n- master (host, idle)\n- researcher (docker, working)")
```

### Сценарий 3: длинная операция

Входящий: `[ag-os chat=249402107 user=Max] собери дайджест по новостям ИБ за сутки`

Это может занять минуту. Отвечай в два хода:

1. Сначала подтверждение:
   ```
   telegram_reply(chat_id=249402107, text="Начинаю собирать дайджест, это займёт 1-2 минуты.")
   ```
2. Делай работу.
3. Финальный ответ с результатом:
   ```
   telegram_reply(chat_id=249402107, text="Готово. Ключевые новости:\n1. ...\n2. ...")
   ```

### Сценарий 4: ошибка при выполнении

Входящий: `[ag-os chat=249402107 user=Max] создай агента без провайдера`

Bash-команда падает с `ValueError: Unknown model provider`. Не молчи — сообщи:

```
telegram_reply(chat_id=249402107, text="Не смог создать агента: не указан model_provider. Какой провайдер использовать из config.model_providers?")
```

## Чего НЕ делать

- **Не отвечай простым текстом без tool.** Пользователь не увидит.
- **Не забывай `chat_id`.** Tool не знает контекста, ему нужен явный id из префикса.
- **Не отправляй чувствительные данные.** API-ключи, токены, содержимое `~/.claude/credentials.json` — никогда в text.
- **Не используй Markdown-форматирование (`**жирный**`, \`код\`) в text.** В этой версии parse_mode не задан, Markdown попадёт как сырые символы. Используй обычные переносы строк, дефисы для списков, эмодзи для визуального разделения.
- **Не шли одно гигантское сообщение длиннее 4096 символов.** Telegram API отрежет. Для длинных — разбивай на части и шли последовательно несколькими вызовами `telegram_reply`.
- **Не игнорируй промты без префикса.** Если ты видишь сообщение без `[ag-os chat=...]` — это не из Telegram (например, пользователь пишет в tmux напрямую или задача от scheduler'а). Отвечай обычно в своей сессии, tool не нужен.

## Взаимодействие с другими скиллами

- Когда пользователь просит создать агента / поставить в cron / записать в vault — сначала делай работу через `agos-agent-management` / `agos-schedule-management` / `agos-vault-curator`, потом возвращай подтверждение через `telegram_reply`.
- `telegram_reply` — это всегда **последний шаг** в цепочке, после того как запрос выполнен.

## Если tool недоступен

Если при попытке вызвать `telegram_reply` ты получаешь ошибку что tool не найден — MCP-сервер не подцепился. Это означает что нужно:
1. Проверить что mcp-сервер зарегистрирован: `claude mcp list`
2. Если нет — переустановить: `claude mcp add ag-os-telegram -- /path/to/.venv/bin/python -m mcp_servers.telegram_bridge`
3. Рестартовать свою сессию claude.

Но сам ты этого сделать не можешь — нужно сообщить пользователю. Как? Если Telegram не работает — пиши в свой pane (пользователь увидит через `tmux attach`). Паника-фраза: `ERROR: MCP telegram tool not available, please re-register via claude mcp add`. Пользователь увидит и починит.
